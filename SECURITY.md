## 报告安全漏洞

如果您发现安全漏洞，请不要公开报告，而是通过以下方式私下联系我们：

**邮箱**: [369752868@qq.com]

请尽可能包含以下信息：
- 漏洞的描述
- 受影响的版本
- 重现漏洞的步骤
- 漏洞的影响和严重程度
- 如果可能，提供修复建议

## 漏洞处理流程

1. **确认接收**：我们会在48小时内确认收到您的报告
2. **调查验证**：我们会调查并验证漏洞
3. **制定修复**：我们会制定修复方案
4. **发布修复**：我们会尽快发布安全更新
5. **公开致谢**：修复发布后，我们会公开致谢（如果您同意）

## 安全最佳实践

### 用户安全建议

1. **保持更新**
   - 及时更新到最新版本
   - 关注安全公告

2. **环境配置**
   - 不要在生产环境中使用默认配置
   - 修改默认端口和密钥
   - 使用HTTPS连接

3. **数据保护**
   - 定期备份聊天数据
   - 保护敏感信息
   - 使用强密码

### 开发者安全建议

1. **依赖管理**
   - 定期更新依赖包
   - 使用`pip-audit`检查已知漏洞
   ```bash
   pip install pip-audit
   pip-audit
   ```

2. **代码安全**
   - 输入验证和输出编码
   - 避免SQL注入
   - 使用参数化查询
   - 敏感数据加密存储

3. **环境变量**
   - 不要在代码中硬编码敏感信息
   - 使用`.env`文件管理环境变量
   - 确保`.env`文件不被提交到版本控制

## 隐私政策

### 数据收集
- 本项目不收集任何用户个人信息
- 聊天数据存储在本地
- 不会上传到任何第三方服务器

### 数据存储
- 聊天记录以JSON格式存储在本地
- 用户设置存储在浏览器的localStorage中
- 所有数据都在用户控制之下

### 数据共享
- 不会与第三方共享用户数据
- 不会出售或租赁用户数据

### 数据删除
- 用户可以随时删除本地数据
- 卸载应用会删除所有相关数据
